<p class="section-label">Overview</p>
<h2 class="section-title">The Three Pillars</h2>
<p class="section-desc">Every configuration lever falls into one of three categories.</p>

<div class="card-grid">
  <div class="card card-accent-blue">
    <h3>📋 Context Management</h3>
    <p>
      <strong>Shape what Copilot knows.</strong> Instructions files, path-based rules,
      and skills give the agent the right context at the right time — so it writes code
      that fits <em>your</em> codebase, not a generic tutorial.
    </p>
  </div>

  <div class="card card-accent-purple">
    <h3>🚧 Deterministic Gates</h3>
    <p>
      <strong>Hard guardrails that can't be prompt-hacked.</strong> Hooks intercept every
      tool call and can block, allow, or modify agent actions — enforcing security, testing,
      and compliance before code ever lands.
    </p>
  </div>

  <div class="card card-accent-green">
    <h3>🔧 Tools &amp; Extensions</h3>
    <p>
      <strong>Extend what Copilot can do.</strong> MCP servers bring external data in;
      CLI extensions add custom tools and governance hooks. Extensions are the unifier —
      they can do context, gates, <em>and</em> tools in one package.
    </p>
  </div>
</div>

<p class="section-label">Motivation</p>
<h2 class="section-title">Why Configure?</h2>
<p class="section-desc">Out-of-the-box Copilot is good. Configured Copilot is transformational.</p>

<div class="card-grid">
  <div class="card">
    <h3>🎯 Adoption at Scale</h3>
    <p>
      Teams that configure instructions and gates see higher trust and faster adoption —
      because the agent already knows the team's patterns on day one.
    </p>
  </div>

  <div class="card">
    <h3>🔒 Governance Without Friction</h3>
    <p>
      Hooks enforce security scanning, test coverage, and code standards automatically —
      no manual review bottleneck, no "the agent forgot to run tests."
    </p>
  </div>

  <div class="card">
    <h3>⚡ Compounding Returns</h3>
    <p>
      Every instruction, extension, and skill you add compounds across every developer
      on the team. One hour of configuration saves hundreds of hours of correction.
    </p>
  </div>
</div>
